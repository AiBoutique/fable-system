# Expertise Atlas — router and system index

The expertise system is 24 domain-mastery skills built on one shared spine: the global operating discipline (CLAUDE.md + `fable-mode`) governs process everywhere; each domain skill adds its domain's evidence sets, non-negotiables, method playbooks, verification ladder, deliverable standards, and boundaries. Every skill guarantees the same contract (below), so composing them is safe: apply the union of whatever applies; where two rules overlap, the stricter governs.

## Routing procedure
1. Classify the request's domain(s) from the directory below — most requests have one owner; engagements often add `consulting-mastery` for method/story/client mechanics.
2. Invoke the owning skill(s). Cross-domain work applies the union; stricter rule wins. Do not invoke more than the request needs — three skills is a normal ceiling for one task.
3. Process rigor is orthogonal: hard, multi-step, or high-stakes work also invokes `fable-mode` regardless of domain (per the global discipline).
4. Format/tooling skills compose freely where the deliverable calls for them: `docx`, `pptx`, `xlsx`, `dataviz`, `deep-research`, and for listed-security research `invest-research` (which owns that workflow end to end).
5. Still ambiguous → open `references/topic-map.md` (all 63 source topics → owner) and pick by topic.

## Shared contract (what every domain skill guarantees)
- Same section order: stance → Compose with the system → Scope of mastery → Evidence set → Non-negotiables → Method → Verification ladder → Deliverables → Boundaries & escalation → References.
- Full verbatim subskill registry in `references/coverage.md` (open for completeness planning or audits).
- Three universal non-negotiables everywhere: cutoff-sensitive facts retrieved with as-of dates, never recalled; decision-steering arithmetic through code/tool; claims labeled verified/inferred/assumed with falsifiers on recommendations.
- Boundaries are binding: licensed-professional lines (clinician, dentist, counsel, advisor), dual-use/CBRN limits, authorized-context security work, analysis-never-advice framing for investments.
- Mastery is enforced by protocol, never asserted: each skill's verification ladder runs before its deliverables ship.

## Common cross-domain pairings
- Any client deliverable → owner skill + `consulting-mastery`. Any hard multi-step task → + `fable-mode`.
- AI initiative: `ai-agentic-systems` (system) + `digital-enterprise-tech` (platform) + `risk-governance-compliance` (governance) as needed.
- Deal: `deals-ma-restructuring` (process) + `finance-value` (model) + domain DD skill (`cyber-trust`, `software-engineering-mastery`, ...).
- Health engagement: health skill (content) + `industry-sector-mastery` (sector economics) + safeguards hosted in `medicine-clinical-health`.
- Pricing change: `commercial-growth` (design) + `data-decision-science` (elasticity/experiments) + `risk-governance-compliance` (algorithmic-pricing fairness).
