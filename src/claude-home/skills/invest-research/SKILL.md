# Investment research playbook

Standing profile (a template — edit to your own): long-term investor; example themes AI, biotech, climate, semiconductors; wants depth and evidence over speed. Hard rule from Rank 0: never write holdings, positions, or amounts to persistent memory.

1. **Scope the tier before gathering.** Casual ask ("thoughts on X?") → quick take: thesis in two sentences, the three numbers that carry it, top risks, the falsifier, all sourced. "Deep dive / analyze / report" → tiered deliverable: executive summary → thesis + scorecard → evidence layers (fundamentals, valuation, ownership, catalysts, risks) → falsifiers and review-by date → appendix with provenance. Ask only when the tier is genuinely ambiguous. Every call names its horizon and benchmark—an "outperform" without both is ungradeable. A ticker or theme the desk has seen before → pull the saved thesis first (get_thesis_scorecard) and open with the delta: prior call, what changed, which assumptions broke.
2. **Data discipline** — the CLAUDE.md economics/finance rules govern in full: every figure carries provenance ([tool: Finviz], [tool: EDGAR], [calc]) and an as-of date; nothing quoted from memory; decimal money math; currency, units, real-vs-nominal, and annualization stated; point-in-time data only for anything backtest-shaped.
3. **Gather multi-angle, not single-query.** When the KnowledgePrime desk is connected, sweep in parallel across independent axes: fundamentals and quality (dossier, deep fundamentals, earnings quality), valuation (relative + scenario models), ownership and insiders, catalysts and filings (EDGAR, risk-factor diffs, filing events), positioning (short interest, dark pool, attention), and macro/sector context. Use web search/news for whatever the desk lacks. No data channel at all → capability PAUSE per CLAUDE.md, never memory-quoting.
4. **Adversarial pass before delivery.** Build the strongest opposing case from the same data — bear case for a bullish read, bull case for a bearish one. The pass also hunts the finance fraud table by name: stale figures presented as current, headline numbers with fees/costs omitted, percentages off the wrong base or mismatched compounding, cherry-picked windows, projections stated as fact, guarantee language. A buy/trim/avoid decision the user actually faces gets the tournament treatment (rubric first, independent judges) per their standing preference; the output is a recommendation with reasoning, never a decision made for them.
5. **Deliver in the requested medium.** Chat → layered summary, headline call first. Report or file requested → gather first, then render via the desk's HTML report tool (never hand-write report HTML). Every call ends with its falsifier and a review-by date. After a deep-dive call, persist it desk-side with save_thesis—the thesis text carries call, horizon, benchmark, and review-by; the falsifier becomes assumptions_json, each a metric+op+threshold the desk can grade (a BROKEN grade = falsifier fired). The desk, never Claude memory, is the thesis store; the holdings rule is untouched. Frame everything as analysis with named risks and assumptions — never as advice, and never as an action taken.
